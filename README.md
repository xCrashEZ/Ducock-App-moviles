Booksy SPA es una app móvil de gestión bibliotecaria pensada para un uso simple y agradable: permite explorar un catálogo por categorías, buscar títulos, ver fichas con autor, sinopsis y disponibilidad, y realizar solicitudes y devoluciones de préstamo simuladas que quedan visibles en el historial del usuario. El proyecto se centra en el alcance del EP3: diseño y experiencia de usuario en móviles, formularios con validaciones claras (por ejemplo, para registro e inicio de sesión), navegación consistente con manejo correcto del backstack, gestión de estado con indicadores de carga, éxito y error, persistencia local para un CRUD básico (favoritos y/o préstamos) y almacenamiento de un avatar, uso de recursos nativos como cámara o galería con permisos y fallback, animaciones con propósito para reforzar acciones clave y consumo de API para autenticación, incluyendo la consulta del perfil con “/auth/me”. Para ejecutar el proyecto basta con clonar el repositorio, abrirlo en Android Studio, sincronizar Gradle y correrlo en un emulador o dispositivo físico; si corresponde, se configura la Base URL del backend en variables locales. A nivel de diseño interno se mantiene una separación de responsabilidades entre interfaz, lógica de presentación y acceso a datos, sin detallar arquitectura MVVM completa ni estructura de carpetas ni implementaciones concretas de Retrofit o Repository, tal como exige la pauta académica; del mismo modo, no se incluye código de validaciones, gráficos de navegación, DAOs de Room, servicios de API ni componentes Compose: el estudiante debe implementarlos. El flujo típico comienza con registro o login, guarda el token y redirige al catálogo; desde ahí el usuario explora, abre un detalle y solicita un préstamo, y puede gestionar su perfil cambiando nombre y foto con cámara o galería. La app contempla escenarios offline mostrando el último catálogo disponible y comunica errores de red con mensajes de reintento. En síntesis, es un proyecto listo para demostrar dominio práctico de UI, navegación, estado, persistencia, recursos nativos, animaciones y consumo de API sin incluir las piezas de código que el alumno debe desarrollar para aprobar.
